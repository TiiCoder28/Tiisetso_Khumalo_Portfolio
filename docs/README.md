# Documentation

This folder is the home for delivery notes and system documentation for the portfolio.

## Structure

- `changes/`: feature-level implementation notes for shipped work.
- `CONTENT_SYSTEM_GUIDE.md` at the repo root: architecture and workflow guide for the content platform.

## Documentation Standard

For each new feature, bug fix, or architecture change, add a new note under `docs/changes/` that records:

- what was added
- what was changed
- what was removed
- what was updated
- why the change was made
- what outcome was achieved
- any known follow-up items

Use dated filenames so the history stays readable.
